# Design System

## Visual Keywords

项目的视觉关键词：

- 慢通信
- 手写信
- 远方
- 海风
- 纸张
- 邮票
- 温柔
- 夜晚
- 旧日记
- 手绘地图

整体氛围应该像一个慢慢展开的信箱，而不是即时通讯工具。

## Color Principles

推荐颜色方向：

- 米白
- 暖黄
- 橄榄绿
- 深棕
- 低饱和蓝灰

避免使用：

- 高饱和科技蓝
- 赛博紫
- 强烈霓虹色
- 过亮的纯色渐变
- 强烈商业 SaaS 风格配色

颜色应该像纸张、旧邮票、植物、木桌、海边傍晚和手绘地图。

## UI Principles

### Cards

卡片应该像信纸或纸片：

- 纸张质感
- 柔和边缘
- 轻微阴影
- 留白充足
- 内容有呼吸感

不要把卡片做成强烈的社交 feed 卡片，也不要做成冷硬的 dashboard 面板。

### Buttons

按钮可以有邮票、信封、封蜡、纸条的感觉。

- 圆角柔和
- 阴影轻
- 状态清楚
- 文字温柔但明确

按钮不应该像强社交软件的高刺激 CTA。

### Layout

页面应该保留较多留白。

- 内容节奏慢
- 模块之间有空气感
- 移动端优先保持阅读舒适
- 不要把页面塞满

不要做强社交软件感，不要做聊天软件感。

## Motion Principles

动效关键词：

- 慢
- 轻
- 淡入
- 漂浮
- 信纸展开
- 地图 pin 缓慢出现

适合的动效：

- 信纸轻轻浮现
- 卡片轻微上浮
- 地图 pin 慢慢出现
- 邮票或信封轻微位移
- 等待状态有非常轻的呼吸感

避免的动效：

- 夸张弹跳
- 快速闪烁
- 强烈缩放
- 游戏化奖励动效
- 高频 loading 动画

动效应该像风吹动纸张，而不是 App 推送提醒。
